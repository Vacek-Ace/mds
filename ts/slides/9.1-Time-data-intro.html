Datos temporales

Introducción

  • Desde finales de 1970 se ha popularizado el análisis de datos con dependencias temporales. Sin embargo, se han desarrollado métodos y herramientas de forma independiente en distintas áreas, sin ninguan cohesión.

  • Solo en las últimas dos décadas se está produciendo un efecto de trasvase de técnicas entre diferentes áreas (economía, demografía, medicina, sociología, ingeniería, etc.).

  • Es importante extremar las precauciones para comprender los requisitos y supuestos que se requieren para aplicar cada técnica, pues de otro modo corremos enormes riesgos a la hora de realizar predicción o inferencia.

    • Muchos métodos estadísticos habituales no pueden aplicarse directamente a datos con correlación temporal entre sí.


  • Curiosamente, en la década de 1960 y 1970 había serias dudas en muchas áreas científicas sobre la posibilidad de desarrollar modelos para medir y comprender el cambio en individuos o grupos, a lo largo del tiempo.

  • Entre 1957 y 1960 aparecieron las primeras técnicas de suavizado exponencial. En 1970 Box y Jenkins publican su influyente libro “Time series analysis: Forecasting and control”, estableciendo las bases para uno de los métodos más populares de análisis de series temporales.

Introducción: datos temporales

  • Los datos temporales son aquellos que ofrecen valores obtenidos a lo largo del tiempo. Es decir, necesitamos un campo (columna, si es una tabla de datos) con información sobre el instante en que fue tomado el dato.

  • En estadística matemática y, especialmente, en economía (econometría) se suele usar el término equivalente datos de panel.

Introducción: datos de panel

  • Decimos que un panel de datos es balanceado cuando tenemos el mismo número de observaciones para cada individuo o caso, en todos los puntos temporales considerados.

  • Si tenemos diferente número de muestras temporales para cada caso (por ejemplo, nos faltan datos para ciertos años, meses, etc.) entonces hablamos de un panel de datos no balanceado.

  • En muchas ocasiones los datos de panel se organizan de la forma en la que hemos visto en la tabla anterior. Ese formato se denomina long format.

  • Otro formato alternativo es poner cada caso o individuo en una sola fila, e ir añadiendo columnas etiquetadas con el nombre de la variable y la fecha o instante temporal en que se tomó ese valor. Este otro formato se denomina wide format.

  • Cómo transformar datos entre long y wide format (RCookbook).

Datos longitudinales y series temporales

  • En ocasiones, se usa también el término datos longitudinales para referirse a datos de panel (con dependencias temporales). Además, tenemos otras series de datos temporales llamadas series temporales.

  • Hay una diferencia entre datos longitudinales y series temporales [Wei 2019].

  • El término datos longitudinales designa a conjuntos de datos que contienen valores para una muestra aleatoria de individuos o casos, registrados en diferentes instantes de tiempo. Para cada caso, tenemos pocos valores en el tiempo.

    • Por ejemplo, un estudio sobre la evolución de resultados académicos de alumnos en un colegio; datos sobre pacientes con problemas cardivasculares, tomados cada 6 meses para cada paciente, etc.
    • El término datos longitudinales proviene de áreas como bioestadística, ciencias de la salud o estudios sociológicos y políticos. A veces, también se usa el término medidas repetidas.


- Una serie temporal designa una única serie de valores (para un solo caso o individuo), medidos a lo largo del tiempo. Normalmente, tenemos muchos valores registrados para cada caso (la serie es “larga”, comparada con datos longitudinales).

Datos longitudinales y series temporales

  • Hay diferencias en los objetivos y métodos de trabajo para cada tipo de análisis.

  • El objetivo en datos longitudinales es analizar diferencias entre individuos o grupos de individuos (o casos), controlando la dependencia temporal entre los datos recogidos.

    • Detectar diferencias entre grupos, controlando la dependencia temporal de los datos para un mismo individuo en diferentes instantes.
    • Por tanto, el principal objetivo en esos estudios suele ser inferencia.


- El objetivo en series temporales es descubrir patrones y estructuras en la serie de datos (aprovechando sus propiedades estocásticas), para proponer un modelo de predicción, que anticipe el valor de futuros datos.

  • En este tema nos ocuparemos de series temporales, estudiando diferentes tipos de modelos que podemos aplicar para estudiar la serie de valores.

Datos transversales y series temporales

  • Entonces, ¿qué aplicación tienen las técnicas estadísticas vistas hasta ahora?

  • Los datos que hemos visto hasta ahora (ignorando dependencias temporales) se denominan en este contexto datos transversales.

    • Datos transversales: Son datos sobre una o varias variables, medidos para varios individuos pero todos ellos obtenidos en el mismo punto temporal. En el panel de datos, correspondería a los datos por filas para varios individuos en el mismo instante de tiempo.

    • Los datos transversales no permiten estudiar cambios en casos o individuos, como tampoco permiten realizar predicción para valores en el futuro (forecasting, diferente de prediction).

    • Series temporales: Una serie temporal se construye como resultado de observar los valores de una variable a lo largo del tiempo, normalmente en intervalos regulares (aunque no es obligatorio, y entonces hablamos de unevenly or irregularly spaced time series). En el panel de datos, correspondería a los datos de una sola variable (columna), medidos para un único individuo o caso.

      • O también hablamos de time-structured data y time-unstructured data [Singer & Willett 2003].

Herramientas para análisis de datos temporales

  • Dependiendo del tipo de pregunta que nos hagamos, podemos elegir entre una amplia variedad de técnicas y herramientas para analizar datos temporales [Singer & Willett, 2003].

    • Comparación de medias entre dos medidas del mismo caso o individuo: paired t-test.
    • Cambios intra-individuales y entre individuos/grupos: mixed effect models o modelos multinivel.
      • Nivel 1: Cambios para un mismo individuo o caso en el tiempo.
      • Nivel 2: Cambios de las diferencias entre individuos o casos en el tiempo.

    • Análisis de tiempo hasta que sucede un evento: análisis de superviviencia.
    • Análisis de secuencias de eventos: modelos multiestado (cadenas de Markov), análisis de secuencia o trayectoria.

Herramientas para análisis de datos temporales

Predicción de valores futuros: análisis de series temporales.

  • Descomposición de estacionalidad y tendencia.
  • Modelos autorregresivos (AR, MA, ARMA, ARIMA, SARIMA, SARUMA, SARIMAX, etc.).
    • Relación entre sistemas dinámicos: funciones de transferencia.
    • Factores que influyen en la serie: análisis de intervención.
  • Modelos para alta volatilidad (ARCH, GARCH).
  • Suavizado exponencial (exponential smoothing).
  • Análisis multivariante de series temporales (en cada instante un vector, no un único valor; e.g. VARMA).
  • Análisis en el dominio de la frecuencia (spectral analysis, transformada de Fourier).
  • Control de procesos (ingeniería, producción).

Nosotros nos vamos a centrar en los dos primeros tipos.

  • Descomposición de estacionalidad y tendencia (STL).
  • Modelos autorregresivos integrados de medias móviles (ARIMA).

Representación de datos temporales

Datos temporales en R

  • Para la representación de datos temporales en R, normalmente distinguimos entre dos posibles situaciones:

    • Análisis de datos de panel: Normalmente se representan mediante un data.frame, siguiendo el formato long. A pesar de ello, ciertas herramientas (por ejemplo, algunos tipos de análisis de secuencia) exigen encontrar los datos en formato wide.

    • Análisis de series temporales: Se suelen emplear objetos (clases) en R especializados para la representación de series temporales.

      • ts ofrece la versión más básica.
      • zoo ofrece soporte para series con espacio regular o irregular, además de soportar también datos faltantes (rupturas en la serie de valores). Intenta ser lo más consistente posible con las funciones del paquete básico ts.
      • xts es un tercer paquete que intenta proporcionar una interfaz uniforme para trabajar con datos de series temporales en R.
      • tsibble es un nuevo paquete del grupo de Rob Hyndman, para representación y análisis de series temporales dentro del ecosistema tidyverse: https://tsibble.tidyverts.org/.
      • tidyquant es otra alternativa para análisis de series temporales, integrada en tidyverse, más enfocado a datos financieros: Introducción a tidyquant.

Ejemplo: conversión long a wide y viceversa

  • Función tidyr::gather(): https://tidyr.tidyverse.org/reference/gather.html.

  • Función tidyr::spread(): https://tidyr.tidyverse.org/reference/spread.html.

  • Nuevas funciones para pivotar: https://tidyr.tidyverse.org/dev/articles/pivot.html.

    • Sustituyen a las antiguas funciones gather() y spread() del paquete tidyr.
    • Incluyen funcionalidades más flexibles y una interfaz teóricamente más intuitiva para transformar los datos entre ambos formatos.

Series temporales en R: xts

  • Ventajas de la clase xts.
    • Existen diferentes clases en R que resuelven varios problemas relacionados con la representación y tratamiento de series temporales.
    • tseries::irts, Rmetrics::timeSeries, paquetes its y zoo.
    • Esto obliga a constantes cambios de clase (casting de variables), en los que se van perdiendo atributos en cada cambio.
    • Además de proporcionar soporte para no perder los atributos propios de cada clase original, xts permite añadir nuevos atributos a cada serie.
  • Basado en la clase zoo.
  • Formatos de fecha soportados: POSIXct, chron, yearmon, yearqtr o timeDate.
  • Desventajas: xts no está integrado con el ecosistema de herramientas y paquetes de tidyverse (nuevo paquetes como tsibble si lo están).

Series temporales en tidyverse

Series temporales y tidy data

  • El conjunto de paquetes que conforman tidyverse se ha vuelto muy popular en R.

  • Implementan los principios de tidy data, introducidos por H. Wickham.

  • Implementan una gramática de datos, similar a la de otros lenguajes de programación (e.g. Pandas en Python) para procesado y análisis de datos estructurados.

  • Incorpora los principios de diseño de bases de datos relacionales para tratar con tablas de datos.

  • Como consecuencia, tiene sentido que aparezcan nuevos paquetes que integren soporte para procesado y análisis de series temporales en este entorno:

    • tsibble es un paquete creado por el equipo de Rob Hyndman (autor, entre otros, del paquete forecast).
    • https://pkg.earo.me/tsibble/.
    • tidyquant es un paquete creado por Matt Dancho, para dar soporte al análisis de series temporales económicas, dentro del ecosistema de paquetes del tidyverse.
    • Vignette: Introduction to tidyquant.

El paquete tsibble

Introducción a tsibble

  • Todavía está en fase de desarrollo (versión 0.8.6 en este momento), pero evoluciona muy rápidamente.

  • Se ha creado un subconjunto de paquetes llamado tidyverts, que integra varias herramientas junto a tsibble para análisis de datos: https://tidyverts.org/.

  • tsibbletiliza la clase tbl_ts para representar tidy data en contextos de análisis temporales. El objeto se construye a partir de una tibble, y consta de:

    • Un índice temporal (una columna de nuestra tabla de datos).
    • Una clave para cada entrada. Puede constar de una o varias columnas, para identificar casos individuales.
    • El resto de variables (columnas) que miden valores en cada instante que indica el índice temporal.

  • Incorpora diversos avances, como por ejemplo soporte para series temporales jerárquicas y agrupadas, esto es, agrupaciones de series temporales relacionadas entre sí (por ejemplo, por género, geografía o tipo de producto).

  • Las series temporales jerárquicas se pueden analizar con el paquete hts de los mismos autores, pero sin soporte para tidy data.

Creación de objetos tsibble

  • La función tsibble() crea un objeto de este tipo.
  • La función as_tsibble() convierte objetos de otro tipo en un tsibble.
    • La conversión es automática para objetos ts, xts o hts.
    • Si es un data.frame o una tibble, necesitamos indicar las columnas que actuarán como índice y clave.
  • Veamos un ejemplo con datos meteorológicos del paquete nycflights13.
weather <- nycflights13::weather %>% 
  select(origin, time_hour, temp, humid, precip)

Creación de objetos tsibble

  • En este caso, partimos de un objeto tibble, así que usamos as_tsibble().
    • Si solo hay una columna con datos temporales, la identifica automáticamente como índice. También se puede indicar explícitamente con el argumento index = col_timestamp.
    • Hay que indicar qué columna queremos que actúe, junto al índice temporal, como clave primaria. En este caso es origin y usamos la función id() para crear automáticamente un identificador entero que indexa de forma eficiente para cada valor de esa columna. El resto de columnas son observaciones.
weather_tsbl <- as_tsibble(weather, key = origin); head(weather_tsbl, 3)

Operaciones con tsibble

  • index_by() es el equivalente de group_by() para la columna temporal. Agrupa los cálculos por intervalos temporales.

  • summarise(), después de index_by(), obtiene resúmenes de datos para cada intervalo temporal.

  • Ejemplo: obtener la temperatura diaria máxima y mínima.

weather_tsbl %>%
  group_by(origin) %>%
  index_by(date = as_date(time_hour)) %>% 
  summarise(
    temp_high = max(temp, na.rm = TRUE),
    temp_low = min(temp, na.rm = TRUE)
  ) %>% head(3)
## # A tsibble: 3 x 4 [1D]

Operaciones con tsibble

  • Existen diversos conjuntos de operaciones con ventanas temporales.

  • slide(), slide2() y pslide() soportan ventanas temporales con solape entre observaciones.

  • tile(), tile2() y ptile() soportan ventanas adosadas, sin solape entre observaciones.

  • strech(), strech2() y pstrech() fijan una ventana inicial que se expande progresivamente, para incluir más observaciones.

  • Importante soportan interfaz e implementación de purrr, por lo que integran principios de paralelización y programación fucnional para hacer más eficientes los cálculos.

  • También se puede elegir realizar el enventanado de datos solo parcialmente, para una parte de la tabla y no para la tabla completa de datos.

  • Información y ejemplos: https://pkg.earo.me/tsibble/articles/window.html.

El paquete tidyquant

Introducción a tidyquant

  • Paquete en versión ya estable (version 1.0.0 en este momento).

  • Implementa herramientas de análisis de series temporales financieras, dentro del entorno tidyverse.

    • xts para soporte de series temporales.
    • quantmod para datos y modelos financieros y bursátiles.
    • TTR para computación de ecuaciones técnicas para operaciones bursátiles.
    • PerformanceAnalytics con funciones econométricas para rendimiento y análisis de riesgos.

  • Los gráficos se construyen a partir de ggplot2, para una mejor integración con el resto de tidyverse y mayor calidad visual.

  • Funciona directamente con una tibble, no hay que crear un objeto especial (como en tsibble).

Conclusiones

  • tsibble y tidyquant son dos nuevos paquetes para procesado y análisis de series temporales en el contexto del entorno tidyverse.

  • De momento, ambos han sido “reconocidos” como los paquetes de referencia para series temporales en este entorno.

    • Max Kuhn tiene planificado incluirlo dentro de la lista de paquetes que carga tidymodels, el metapaquete para unificar las diferentes herramientas para modelado de datos y análisis estadístico en tidyverse.
    • https://www.tidyverse.org/articles/2018/08/tidymodels-0-0-1/.
    • https://github.com/tidymodels/tidymodels.

  • tsibble es más potente, ya que incluye soporte para definir claves primarias, que van a permitir cruzar y alinear datos de diferentes series temporales (análisis multivariante), para series jerárquicas y/o agrupadas.

  • tidyquant es más sencillo, ya que opera directamente sobre objetos tibble, detectando la columna que contiene el índice temporal. Utiliza las mismas funciones que xts, quantmod y PerformanceAnalytics, añadiendo soporte para tidy data.
